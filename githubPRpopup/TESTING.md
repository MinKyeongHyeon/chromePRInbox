# PR Inbox — TESTING

작성일: 2025-12-24

## 목적

수동으로 주요 기능(스누즈, 알림, 배지, 필터 등)을 검증하기 위한 체크리스트와 재현 방법을 정리합니다.

## 빠른 체크리스트

- [ ] `manifest.json` 권한 확인: `storage`, `alarms`, `notifications` 포함
- [ ] 확장 로드: chrome://extensions 에서 개발자 모드로 로드
- [ ] 토큰 설정: Options 페이지에서 GitHub 토큰 입력

## 스누즈 기능

1. 팝업에서 PR 카드의 `SNOOZE` 버튼을 클릭하여 1분(수동 입력)으로 스누즈 설정
2. 확장 백그라운드에서 알람 예약 확인:
   - 콘솔에서 `chrome.alarms.getAll` 호출 또는 서비스워커 로그 확인
3. 만료 시 데스크탑 알림 수신 확인
4. 알림 클릭 시 PR이 하나의 탭으로 열리거나 기존 탭이 포커스되는지 확인
5. 팝업의 `스누즈 관리` 패널에서 항목 목록, 검색, 정렬, `열기`, `언스누즈`, `모두 해제` 동작 확인

## 배지 업데이트

- [ ] PR 목록에서 스누즈된 항목은 배지 숫자에서 제외되는지 확인
- [ ] PR을 열어 `seenPRs`로 저장되면 배지 숫자가 감소하는지 확인

## 알림/클릭 일관성

- [ ] 동일 PR에 대해 여러 번 스누즈/만료를 반복해도 알림 클릭 시 탭이 중복 생성되지 않는지 확인

## 네트워크/폴백

- [ ] Notifications API로 PR 후보가 적절히 로드되는지 확인
- [ ] authored-search 또는 GraphQL 폴백 동작 확인(토큰 스코프가 충분한지 확인)

## 오류 재현 및 로깅

- 서비스 워커(확장 배경 콘솔)를 열고 에러 로그가 없는지 확인
- 네트워크 오류는 팝업의 진단 버튼(`진단`)으로 상세 응답을 확인

## 메모

- GitHub 토큰 권한(scope)에 따라 일부 PR/라벨/리포 정보에 접근 불가할 수 있음.
- 데스크탑 알림 동작은 운영체제/Chrome 알림 권한에 따라 달라질 수 있음.
