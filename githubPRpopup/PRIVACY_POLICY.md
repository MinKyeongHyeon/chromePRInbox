PR Inbox — 개인정보 처리방침

마지막 업데이트: 2025-12-24

요약

- 이 확장 프로그램은 사용자의 GitHub Personal Access Token(PAT)을 사용하여 GitHub 알림 및 PR 정보를 로컬에서 가져옵니다.
- 사용자의 토큰은 기본적으로 브라우저의 `chrome.storage.local` 또는 `chrome.storage.sync`에 저장됩니다. 사용자가 선택하면 토큰을 로컬에서 암호화하여 저장할 수 있습니다.

수집 항목

- 필수: GitHub Personal Access Token (사용자 직접 입력)
- 선택적: 사용자 표시 이름, 아바타(깃허브에서 조회), 확장 기능 설정(테마, 필터 등)
- 내부 수집: PR 캐시, 본문 메타데이터(제목, URL, 레포명), 스누즈/핀/seen 상태

목적

- GitHub API 호출을 통해 사용자의 알림·PR 목록을 조회하고, 확장 내부 UI에 보여주기 위함
- 스누즈 알림을 예약하고 로컬 알림을 생성하기 위함

저장 및 보안

- 토큰 저장: `chrome.storage.local`(기본) 또는 사용자가 선택한 경우 `chrome.storage.sync`(암호화 권장)
- 암호화: Web Crypto 기반 암호화(비밀번호 유도 PBKDF2 + AES-GCM)를 통해 사용자가 선택 시 토큰을 암호화하여 저장합니다.
- 서버 전송: 기본 기능은 서버 전송을 하지 않습니다. 사용자가 명시적으로 서버 기능(예: 계정 동기화)을 활성화하지 않는 한 토큰/민감데이터는 외부 전송되지 않습니다.

제3자 서비스

- 결제/구독 기능을 추가할 경우 Stripe, Gumroad, Paddle 등의 결제 제공업체를 사용할 수 있으며, 이 경우 결제 관련 정보는 해당 업체를 통해 처리됩니다.

데이터 삭제 및 권리

- 사용자는 확장 옵션에서 언제든지 토큰 및 로컬에 저장된 데이터(캐시, snoozes, seen, pinned, savedViews 등)를 삭제할 수 있습니다.
