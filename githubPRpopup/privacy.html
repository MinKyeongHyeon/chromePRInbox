Privacy Policy — PR Inbox

Last updated: 2025-12-26

Summary

PR Inbox uses your GitHub token only to query GitHub APIs for unread pull request notifications and related metadata. The token and any other data used by the extension (settings, cached metadata, snooze/pin state) are stored locally in your browser using chrome.storage and are never transmitted to external servers or third parties.

Data We Collect and Store

  • GitHub token — Provided directly by the user via the extension Options. Stored in chrome.storage.sync to persist across the user's Chrome profile devices (if sync is enabled).
  • User preferences — UI language, filters, auto-refresh interval, theme, and similar settings (stored locally to keep a consistent experience).
  • Runtime cache and state — Cached PR metadata, seen/seen-pr lists, snooze and pinned PR state, and small caches to reduce API requests. These are stored locally and expire based on the extension's caching rules.

How We Use the Data

The data is used solely to provide the extension's functionality: to query GitHub for the user's unread PR notifications, display them in the popup, allow filtering, pinning, and snoozing behavior, and to show local reminders when a snooze expires. No data is used for advertising, profiling, or shared with third parties.

Remote Code and External Requests

PR Inbox does not download or execute remote scripts. The extension's executable code is included in the distributed package. Network requests are limited to calls to the GitHub API ( https://api.github.com/*) for the explicit purpose of retrieving PR and notification data.

Permissions

  • Host permission (https://api.github.com/*): required to call GitHub APIs.
  • storage: to store token and user settings locally.
  • alarms: to schedule local snooze expirations and optional refreshes.
  • notifications: to show local desktop notifications when snoozes expire or notable updates occur.

User Controls

Users can remove or replace their GitHub token at any time from the extension Options page. Deleting the token prevents further API calls. Users may also clear other stored data (filters, snoozes, pinned items, caches) using the Options page.
